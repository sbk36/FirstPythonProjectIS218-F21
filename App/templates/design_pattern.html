Command Design Pattern

Command is a behavioral design pattern that converts a request into a stand-alone object that contains all of the request's information. This transformation allows you to pass requests as method arguments, delay or queue the execution of a request, and support undoable operations.Lets see an example,

To Follow along you can reference the code here -> Github

Here we are using command design pattern by sending commands to the calculations library by doing what it needs to do.Like add subtraction, add multiplication etc.

The advantage of command design pattern is that:-

    It separates the classes that invoke the operation from the object that knows how to carry it out.
    It provides a queue system that allows you to create a sequence of commands.
    The addition of a new command is simple and can be accomplished without modifying the existing code.
    The Command pattern can also be used to define a rollback system; for example, in the Wizard example, we could write a rollback method.
